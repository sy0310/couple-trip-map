# External Integrations

**Analysis Date:** 2026-04-20

## APIs & External Services

**Supabase (Primary Backend):**
- Database: PostgreSQL via Supabase REST API
  - SDK: `@supabase/supabase-js` 2.103.3
  - Auth: Supabase Auth (email/password, magic links)
  - Storage: Supabase Storage bucket (`photos`)
  - Connection: `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- SSR Support: `@supabase/ssr` 0.10.2 — Cookie-based session management in Next.js middleware

**OpenStreetMap (Map Tiles):**
- Tile Layer: `https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png`
- Used in: `src/components/leaflet-map.tsx`
- Free, no API key required

**WeChat Cloud (Mini Program — separate codebase):**
- Cloud functions: `cloudfunctions/couple/`, `cloudfunctions/trip/`, `cloudfunctions/photo/`, `cloudfunctions/login/`
- Environment ID placeholder: `your-env-id` in `miniprogram/app.js`

## Data Storage

**Database:**
- PostgreSQL via Supabase
- Tables: `users`, `couples`, `trips`, `locations`, `photos`
- Migration: `migrations/001_initial_schema.sql`
- TypeScript types: `src/lib/database.types.ts` (generated by Supabase)
- RLS (Row Level Security): Enabled on all tables

**File Storage:**
- Supabase Storage bucket: `photos`
- Path format: `{coupleId}/{uuid}.{ext}`
- Public URL access for display

**Caching:**
- None — no Redis or external cache layer

## Authentication & Identity

**Auth Provider:**
- Supabase Auth
  - Email/password + magic link authentication
  - SSR cookie session via `@supabase/ssr`
  - Middleware: `src/middleware.ts` — calls `supabase.auth.getUser()` on every request
  - Auth callback: `src/app/auth/callback/page.tsx`
  - Login page: `src/app/login/page.tsx`
  - Auth helper: `src/lib/auth.ts`
